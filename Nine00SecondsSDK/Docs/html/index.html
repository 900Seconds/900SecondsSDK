Nine00SecondsSDK Reference

How to install Nine00SecondsSDK library to your project and set dependencies

Nine00SecondsSDK uses several third-party libraries in its work. In order to use it in your project you will need to add those libraries too. These are steps to set all required dependecies:

  1. Nine00SecondsSDK requires following list of libraries

    • AFNetworking (github)
    • AWSiOSSDKv2/S3 (v2.0.8) (github)
    • CocoaLumberjack (github)
    • THObserversAndBinders (github)
    • ffmpeg (github) - only requires .a static libraries avcodec.a, avdevice.a, avformat.a, avutil.a
    • libextobjc (github)

    Some of the listed libraries (like AWSiOSSDKv2/S3) have their own requirements that have to be set too. Alternatively you can add these listed dependencies using the CocoaPods. It will automatically add all required libraries and frameworks.

    If you want to be able to run you project on simulator then you also have to add iconv system library to it:

    • libiconv.dylib

    Simply add it in project settings inside “Link Frameworks and Libraries” section.

  2. To add Nine00SecondsSDK to the project you need to add

    • Nine00SecondsSDK.a library. It is compiled for both iPhone and iPhone Simulator architectures. So it can be used for both.
    • All NHS-prefixed header files that comes with it

  3. To use the SDK classes you need to import Nine00SecondsSDK.h file:

    #import "Nine00SecondsSDK.h"

    Or you can add headers for specific classes that you want to use:

    #import "NHSBroadcastManager.h"
